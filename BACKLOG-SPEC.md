## 전제 조건

- `~/.claude/learning-backlog.md` 파일이 존재해야 한다.
- 글로벌 CLAUDE.md의 "지식 내재화 하네스"에 의해 세션 중 학습 포인트가 기록된다.

## 백로그 형식

마크다운 테이블로, 한 행이 하나의 학습 포인트이다. 파일 첫 부분에 헤딩과 설명이 있고, 그 뒤에 테이블이 온다.

| 열 | 설명 | 허용 값 |
|----|------|---------|
| 날짜 | 항목이 기록된 날짜 | `YYYY-MM-DD` |
| 프로젝트 | cwd의 마지막 경로 컴포넌트 | 자유 텍스트 |
| 개념 | 독립 학습이 필요한 개념 | 자유 텍스트 (`\|` 문자 사용 금지) |
| 깊이 | 도메인 깊이 수준 | `전공` / `학습중` / `교양` |
| 시작점 | 왜 더 알아야 하는지, 어디서 시작하면 좋을지 | 자유 텍스트 (`\|` 문자 사용 금지) |
| 상태 | 소비 여부 | 빈 문자열 (미소비) 또는 `done` (소비 완료) |

```markdown
# Learning Backlog

세션 중 표면적으로만 다룬 개념 중, 독립 학습이 권장되는 항목의 누적 기록.

| 날짜 | 프로젝트 | 개념 | 깊이 | 시작점 | 상태 |
|------|----------|------|------|--------|------|
| 2026-04-10 | my-ml-project | NCCL ring allreduce 토폴로지 최적화 | 전공 | NCCL docs topology detection 섹션 | |
| 2026-04-11 | k8s-setup | CNI 플러그인 성능 차이 | 학습중 | Cilium 벤치마크 문서 | done |
```

## 파싱 규칙

- 테이블 시작: `| 날짜 |`로 시작하는 행이 헤더이다.
- 헤더 바로 다음 행은 구분선(`|------|...`)이다. 두 행 모두 스킵한다.
- 이후 각 행을 `|`로 split하여 양쪽 공백을 trim한다. 좌우 끝의 빈 요소를 제거하면 6개 열이 남는다.
- `상태` 열이 비어있으면 미소비, `done`이면 소비 완료로 판단한다.
- 빈 줄이나 테이블 형식이 아닌 줄은 무시한다.
- 파일이 없거나 파싱에 실패하면 해당 섹션을 생략한다.
