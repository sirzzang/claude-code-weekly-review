# Weekly Review Skill

Claude Code 세션 로그(`~/.claude/session-logs/*.jsonl`)를 분석하여 주간 사용 패턴을 회고하고, 프롬프트 품질 개선점을 도출한다.

## 전제 조건

- Stop hook(`~/.claude/hooks/log-session.py`)이 설정되어 있어야 한다.
- 로그 파일은 `~/.claude/session-logs/YYYY-MM-DD.jsonl` 형식으로 존재한다.

## 로그 형식

각 줄은 하나의 세션 기록이며 아래 필드를 포함한다:

```json
{
  "timestamp": "ISO 8601",
  "date": "YYYY-MM-DD",
  "session_id": "string",
  "cwd": "작업 디렉토리",
  "prompts": ["사용자 프롬프트 목록"],
  "turn_count": 3,
  "tool_usage": {"Bash": 5, "Read": 3, "Edit": 2},
  "tool_total": 10,
  "elapsed_seconds": 180,
  "elapsed_human": "3m 0s"
}
```

## 실행 절차

### 1단계: 로그 수집

```bash
# 최근 7일간 로그 파일 확인
ls -la ~/.claude/session-logs/ | tail -10

# 최근 7일 로그를 합쳐서 읽기
find ~/.claude/session-logs/ -name "*.jsonl" -mtime -7 -exec cat {} + 2>/dev/null
```

로그가 없으면 사용자에게 Stop hook 설정 여부를 확인하고, 아직 데이터가 쌓이지 않았을 가능성을 안내한다.

### 2단계: 정량 분석

아래 지표를 계산한다:

- **총 세션 수**: 기간 내 세션 개수
- **총 프롬프트 수**: 전체 turn_count 합계
- **평균 세션 소요 시간**: elapsed_seconds의 평균
- **평균 턴 수**: 세션당 평균 프롬프트 횟수
- **도구 사용 분포**: 도구별 총 사용 횟수와 비율
- **프로젝트 분포**: cwd 기준으로 어떤 프로젝트에서 작업했는지
- **일별 활동량**: 날짜별 세션 수 추이

### 3단계: 프롬프트 품질 분석

수집된 프롬프트를 아래 기준으로 평가한다. 전수 조사하지 말고, 패턴별로 대표 사례를 뽑아서 분석한다.

#### 평가 기준

| 기준 | Good | Needs Improvement |
|------|------|-------------------|
| 구체성 | 파일 경로, 함수명, 에러 메시지를 명시 | "이거 고쳐줘", "에러 나는데" |
| 맥락 제공 | 배경, 제약 조건, 기대 결과를 설명 | 맥락 없이 코드만 던짐 |
| 범위 설정 | 한 세션에 하나의 명확한 목표 | 여러 관계없는 작업을 한 세션에서 처리 |
| 출력 형식 지정 | 원하는 형식을 명시 (테스트 포함, Go 코드 등) | 형식 미지정 후 수정 요청 반복 |
| 반복 패턴 | 한 번에 원하는 결과 도출 | 같은 의도의 프롬프트를 여러 번 재시도 |

#### 분석 방법

1. 턴 수가 비정상적으로 많은 세션(상위 20%)을 식별한다. 프롬프트가 불명확해서 반복한 경우가 많다.
2. 한 줄짜리 짧은 프롬프트를 모아서 패턴을 확인한다.
3. 도구 사용이 과도하게 많은 세션을 확인한다. 범위가 너무 넓었을 가능성이 있다.
4. 잘 작성된 프롬프트 사례도 함께 제시하여 균형을 맞춘다.

### 4단계: 리포트 작성

아래 구조로 회고 리포트를 작성한다. 마크다운 형식으로 출력한다.

```
# 주간 Claude Code 회고: {시작일} ~ {종료일}

## 요약 지표
(정량 분석 결과를 간결하게)

## 프로젝트별 활동
(cwd 기준 분류)

## 도구 사용 패턴
(도구별 빈도와 특이 패턴)

## 프롬프트 품질 분석

### 잘한 점
(구체적인 프롬프트 예시와 함께)

### 개선할 점
(구체적인 프롬프트 예시 + 개선된 버전 제안)

## 이번 주 핵심 개선 포인트
(실행 가능한 1-3개 액션 아이템)
```

### 5단계: 기간 커스텀

사용자가 특정 기간을 지정하면 해당 기간의 로그만 필터링한다:

```bash
# 예: 2026-03-17부터 2026-03-23까지
find ~/.claude/session-logs/ -name "*.jsonl" | sort | while read f; do
  date_part=$(basename "$f" .jsonl)
  if [[ "$date_part" >= "2026-03-17" && "$date_part" <= "2026-03-23" ]]; then
    cat "$f"
  fi
done
```

## 주의사항

- 프롬프트 원문에는 민감 정보(API 키, 비밀번호 등)가 포함될 수 있다. 리포트에 원문을 인용할 때 민감 정보는 마스킹한다.
- 비판이 아닌 개선 제안의 톤을 유지한다. 목표는 프롬프팅 습관의 점진적 향상이다.
- 데이터가 3일 미만이면 주간 회고 대신 현재까지의 스냅샷으로 제시한다.
- Stop hook 특성상 사용자가 Ctrl+C로 강제 종료한 세션은 기록되지 않을 수 있다.
